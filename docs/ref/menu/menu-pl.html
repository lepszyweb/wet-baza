Menu

Pytania lub komentarze?

Cel

Zapewnia interaktywne menu z opcjonalnymi podmenu. Implementuje wzorzec projektowy menu WAI-ARIA .

Jak wdrożyć

  1. Dodaj następujący kod do obszaru header strony. Uwaga: ten krok należy już wykonać w przypadku większości motywów.
    • Strony polskie: 
      <nav role="navigation" id="wb-sm" data-trgt="mb-pnl" class="wb-menu visible-md visible-lg" typeof="SiteNavigationElement">
	<div class="container nvbar">
		<h2>Menu tematyczne</h2>
		<div class="row">
			<ul class="list-inline menu">
				<li><a href="#">Sekcja 1</a></li>
				<li><a href="#">Sekcja 2</a></li>
				<li><a href="#">Sekcja 3</a></li>
			</ul>
		</div>
	</div>
</nav>
    • Strony angielskie: 
      <nav role="navigation" id="wb-sm" data-trgt="mb-pnl" class="wb-menu visible-md visible-lg" typeof="SiteNavigationElement">
	<div class="container nvbar">
		<h2>Topics menu</h2>
		<div class="row">
			<ul class="list-inline menu">
				<li><a href="#">Section 1</a></li>
				<li><a href="#">Section 2</a></li>
				<li><a href="#">Section 3</a></li>
			</ul>
		</div>
	</div>
</nav>
  2. Zaktualizuj element ul za pomocą elementów listy (li) zawierających pożądane łącza.
  3. Opcjonalnie: Utwórz pełne menu w AJAX. To menu zastąpi element class="container" i jego potomków. Podmenu uzyskuje się przez zagnieżdżenie <ul class="sm list-unstyled" id="lore" role="menu"> w każdym elemencie li paska menu. Oto przykład pełnego menu:
    • Strony polskie: 
      <div class="pnl-strt container visible-md visible-lg nvbar">
	<h2>Menu tematyczne</h2>
	<div class="row">
		<ul class="list-inline menu" role="menubar">
			<li><a href="#lore" class="item">Sekcja 1</a>
				<ul class="sm list-unstyled" id="lore" role="menu">
					<li><a href="#">Artykuł 1.1</a></li>
					...
				</ul>
			</li>
			<li><a href="#lor" class="item">Sekcja 2</a>
				<ul class="sm list-unstyled" id="lor" role="menu" >
					<li><a href="#">Artykuł 2.1</a></li>
					...
					<li class="slflnk"><a href="#">Sekcja 2 - Więcej</a></li>
				</ul>
			</li>
			<li><a href="#lorm" class="item">Sekcja 3</a>
				<ul class="sm list-unstyled" id="lorm" role="menu" >
					<li>
						<details>
							<summary>Sekcja 3.1</summary>
							<ul role="menu">
								<li><a href="#">Artykuł 3.1.1</a></li>
								...
							</ul>
						</details>
					</li>
					...
				</ul>
			</li>
		</ul>
	</div>
</div>
    • Strony angielskie: 
      <div class="pnl-strt container visible-md visible-lg nvbar">
	<h2>Topics menu</h2>
	<div class="row">
		<ul class="list-inline menu" role="menubar">
			<li><a href="#lore" class="item">Section 1</a>
				<ul class="sm list-unstyled" id="lore" role="menu">
					<li><a href="#">Item 1.1</a></li>
					...
				</ul>
			</li>
			<li><a href="#lor" class="item">Section 2</a>
				<ul class="sm list-unstyled" id="lor" role="menu" >
					<li><a href="#">Item 2.1</a></li>
					...
					<li class="slflnk"><a href="#">Section 2 - More</a></li>
				</ul>
			</li>
			<li><a href="#lorm" class="item">Section 3</a>
				<ul class="sm list-unstyled" id="lorm" role="menu" >
					<li>
						<details>
							<summary>Section 3.1</summary>
							<ul role="menu">
								<li><a href="#">Item 3.1.1</a></li>
								...
							</ul>
						</details>
					</li>
					...
				</ul>
			</li>
		</ul>
	</div>
</div>
  4. Opcjonalnie: Dodaj podmenu do pełnego menu za pomocą zagnieżdżonych elementów details.
    Ostrzeżenie: podmenu znacznie zwiększają złożoność menu, szczególnie w czytnikach ekranu i urządzeniach mobilnych, co może powodować problemy z użytecznością dla niektórych użytkowników. 
    <ul class="sm list-unstyled" id="lorm" role="menu" >
	<li>
		<details>
			<summary>Sekcja 3.1</summary>
			<ul role="menu">
				<li><a href="#">Artykuł 3.1.1</a></li>
					...
			</ul>
		</details>
	</li>
	...
</ul>
  5. Opcjonalnie: Dodaj data-ajax-replace="[url]" do elementu za pomocą class="wb-menu", z [url] będąc adresem URL pełnego menu dla AJAX w. 
    <nav role="navigation" id="wb-sm" data-ajax-replace="ajax/menu-include-en.html" data-trgt="mb-pnl" class="wb-menu visible-md visible-lg" typeof="SiteNavigationElement">
  6. Opcjonalnie: Zaktualizuj data-trgt za pomocą id elementu docelowego dla panelu mobilnego (w większości przypadków nie powinno się to zmienić).

Menu mobilne włączone do stopki

W przypadku małych i średnich ekranów menu jest renderowane jako panel boczny, uruchamiany przyciskiem menu w nagłówku i zawiera menu dodatkowe oraz stopkę. Sekcje stopki muszą być zgodne z obsługiwanymi wzorami projektowymi, aby były renderowane w menu bez problemu.

Obsługiwane

Nagłówek i lista łączy

Cel

Użyj nagłówka, a następnie listy, gdy trzeba wyświetlić wiele łączy. W panelu menu jest to renderowane jako element <details>, a nagłówek jako podsumowanie.

Kod

<section class="col-sm-4">
	<h3>Nagłówek 1</h3>
	<ul class="list-unstyled">
		<li><a href="#">Element listy 1.1</a></li>
		<li><a href="#">Element listy 1.2</a></li>
	</ul>
</section>
Nagłówek z pojedynczym łączem

Cel

Nagłówek, po którym następuje pojedyncze łącze, samodzielnie lub w akapicie. W panelu menu jest on renderowany jako pojedynczy punkt menu, z nagłówkiem jako tekstem.

Kod

<section class="col-sm-4">
	<h3>Nagłówek 1</h3>
	<p><a href="#">Łącze 1</a></p>
</section>
Nagłówek łaczem

Cel

Nagłówek sam w sobie, zawierający link. W panelu menu jest to renderowane jako pojedynczy element menu

Kod

<section class="col-sm-4">
	<h3><a href="#">Nagłówek 1</a></h3>
</section>

Nieobsługiwane

Nagłówek - łącze, po którym następuje wykaz łączy

Cel

Nagłówek, po którym następuje wiele łączy, samodzielnie, w akapitach lub w mniejszych nagłówkach. Tylko pierwszy link będzie klikalny w panelu menu.

Kod

<section class="col-sm-4">
	<h3><a href="#">Nagłówek 1</a></h3>
	<ul class="list-unstyled">
		<li><a href="#">Element listy 1.1</a></li>
		<li><a href="#">Element listy 1.2</a></li>
	</ul>
</section>
Wiele nagłówków, każdy zawierający łącza

Cel

Wiele nagłówków tego samego rozmiaru, każdy zawierający łącza. Spowoduje to niechciane przerwy w panelu menu. Ponadto ta struktura nie jest zgodna ze specyfikacją HTML5.

Kod

<section class="col-sm-4">
	<h3>Nagłówek 1</h3>
	<p><a href="#">Łącze 1.1</a></p>
	<p><a href="#">Łącze 1.2</a></p>
</section>
Wiele nagłówków - łączy

Cel

Wiele nagłówków tej samej wielkości, z których każdy zawiera linki. Spowoduje to powstanie niepożądanych luk w panelu menu. Dodatkowo, struktura ta nie jest zgodna ze specyfikacją HTML5.

Kod

<section class="col-sm-4">
	<h3><a href="#">Nagłówek 1</a></h3>
	<h3><a href="#">Nagłówek 2</a></h3>
</section>

Bądź ostrożny

Mieszanie różnych obsługiwanych struktur będzie działać, ale lepiej zachować spójność i używać tylko jednej.

Opcje konfiguracyjne

Opcja Opis Jak skonfigurować Wartości
data-ajax-replace Używany do AJAX-a w pełnym menu. Dodaj data-ajax-replace="[url]" do elementu za pomocą class="wb-menu", z [url] z adresem URL pełnego menu do AJAX.
Brak (domyślnie):
Pełne menu nie jest obsługuwane w AJAX.
URL
URL dla pełnego menu do AJAX.
data-trgt Używany do identyfikacji celu dla przenośnego panelu. Dodaj data-trgt="mb-pnl" do elementu za pomocą class="wb-menu". Wartość data-trgt musi pasować do id elementu docelowego dla panelu przenośnego.
Ciąg
Atrybut id elementu docelowego (powinien pozostać mb-pnl chyba że idzostało zmienione.

Zdarzenia

Zdarzenie Wyzwalacz Co robi
wb-init.wb-menu Wyzwalane ręcznie (e.g., $( ".wb-menu" ).trigger( "wb-init.wb-menu" );). Służy do ręcznej inicjalizacji wtyczki menu. Uwaga: Wtyczka menu zostanie zainicjowana automatycznie, chyba że znacznik menu zostanie dodany po załadowaniu strony.
wb-ready.wb-menu (v4.0.5+) Wyzwalane automatycznie po zainicjowaniu menu. Służy do określania, które menu zostało zainicjowane (cel zdarzenia). 
$( document ).on( "wb-ready.wb-menu", ".wb-menu", function( event, instance, settings ) {
});
$( ".wb-menu" ).on( "wb-ready.wb-menu", function( event ) {
});
wb-ready.wb (v4.0.5+) Wyzwalane automatycznie, gdy WET zakończy ładowanie i wykonywanie. Służy do określania, kiedy wszystkie wtyczki WET i polifill zakończyły ładowanie i uruchamianie. 
$( document ).on( "wb-ready.wb", function( event ) {
});
